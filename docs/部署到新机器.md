# 部署到新机器（完整操作手册）

以从开发机（azureuser@riki-model）部署到目标机（root@liteserver-fb87）为例。

---

## 前置条件

- 目标机已安装 Docker 或 Podman
- 目标机有 GPU 并已安装 NVIDIA 驱动
- 目标机已部署 vLLM 模型服务（或准备部署）
- 两台机器之间网络可达（可 SSH/SCP）

---

## 第一步：在开发机上打包

### 1.1 打包代码

```bash
cd /home/azureuser/workspace
tar czf tag-deploy.tar.gz \
  --exclude='tag/.git' \
  --exclude='tag/result' \
  --exclude='tag/data/report_new' \
  --exclude='tag/__pycache__' \
  --exclude='tag/.venv' \
  --exclude='tag/result.tar.gz' \
  tag/
```

### 1.2 构建 Docker 开发镜像

使用 `docker/Dockerfile.dev`（只装依赖，代码通过 volume 挂载）：

```bash
cd /home/azureuser/workspace/tag
docker build -t tag-extract-dev:latest -f docker/Dockerfile.dev .
```

> 如果 Docker Hub 拉不到 `python:3.11-slim`，先配国内镜像源（见附录）。

### 1.3 导出镜像

```bash
docker save tag-extract-dev:latest | gzip > tag-extract-dev.tar.gz
```

---

## 第二步：传输到目标机

```bash
# 传代码包（几 MB）
scp tag-deploy.tar.gz root@<目标机IP>:/root/riki_workspace/

# 传镜像（几百 MB）
scp tag-extract-dev.tar.gz root@<目标机IP>:/root/riki_workspace/

# 传数据（如果目标机上还没有）
rsync -avz --progress /path/to/data/ root@<目标机IP>:/root/riki_workspace/data/

# 传模型（如果需要在目标机上部署 vLLM）
rsync -avz --progress /home/azureuser/models/Qwen3-VL-32B-Instruct-FP8/ \
  root@<目标机IP>:/root/riki_workspace/models/Qwen3-VL-32B-Instruct-FP8/
```

---

## 第三步：在目标机上部署

### 3.1 解压代码

```bash
cd /root/riki_workspace
tar xzf tag-deploy.tar.gz
```

目录结构：

```
/root/riki_workspace/
├── tag/          # 代码
├── data/         # PDF 数据（按 YYYY-MM/类型 组织）
├── models/       # 模型文件（如需本机部署 vLLM）
└── result/       # 抽取结果（在 tag/result 下）
```

### 3.2 加载 Docker 镜像

Docker：

```bash
docker load < tag-extract-dev.tar.gz
```

Podman（如果镜像名变成 `localhost/latest:latest`，需要重新打标签）：

```bash
podman load -i tag-extract-dev.tar.gz

# 如果输出 "Loaded image(s): localhost/latest:latest"，打标签：
podman tag localhost/latest:latest tag-extract-dev:latest
```

### 3.3 修改 config.yaml

编辑 `/root/riki_workspace/tag/config.yaml`，修改以下内容：

#### 数据路径（使用容器内路径）

```yaml
paths:
  data_root: "/data"
  result_root: "/app/result"
  kg_data_root: "/data"
```

#### VL 模型地址

先查看目标机上有哪些 vLLM 实例：

```bash
ps aux | grep vllm | grep -v grep
```

查看模型名和 context 长度：

```bash
curl http://localhost:<端口>/v1/models
```

然后修改 config：

```yaml
vl:
  api_urls:
    - url: "http://localhost:<端口1>/v1/chat/completions"
      model: "<实际模型名>"
    - url: "http://localhost:<端口2>/v1/chat/completions"
      model: "<实际模型名>"
  model: "<实际模型名>"
  context_length: 65536    # 与 vLLM --max-model-len 一致
  max_tokens: 8192
  parallel_workers: 2      # 有几个 vLLM 实例写几
```

> **注意**：model 名必须与 vLLM 启动时的 `--served-model-name` 完全一致。

#### LLM 地址（如有使用）

确认 `llm.chat.api_base` 等地址在目标机上可访问，否则改为可用的地址或云端 API。

### 3.4 启动容器

Docker：

```bash
cd /root/riki_workspace/tag
docker run -d --name tag-extract \
  --network host \
  -v $(pwd):/app \
  -v /root/riki_workspace/data:/data \
  --restart unless-stopped \
  tag-extract-dev:latest
```

Podman：

```bash
cd /root/riki_workspace/tag
podman run -d --name tag-extract \
  --network host \
  -v $(pwd):/app \
  -v /root/riki_workspace/data:/data \
  --restart unless-stopped \
  localhost/tag-extract-dev:latest
```

> `--network host` 让容器直接使用宿主机网络，访问 `localhost:8003` 即为宿主机上的 vLLM。

### 3.5 验证

```bash
# 服务健康检查
curl http://localhost:8013/health

# VL 模型健康检查
curl http://localhost:8003/health

# 测试抽取（先跑 1 个文件）
podman exec -it tag-extract bash
python scripts/extract_kg.py --type meeting_materials --start 2025-03 --limit 1
```

## 第四步：运行抽取任务

### 用 screen 后台运行（防止 SSH 断开中断）

```bash
# 创建 screen 会话
screen -S extract

# 进容器
podman exec -it tag-extract bash

# 运行抽取
python scripts/extract_kg.py --type periodic_report --start 2025-01 --end 2025-03

# 按 Ctrl+A 然后按 D 脱离会话（脚本继续后台运行）
```

### 常用抽取命令

```bash
# 按类型抽取
python scripts/extract_kg.py --type periodic_report --start 2025-01
python scripts/extract_kg.py --type esg_report --start 2025-01
python scripts/extract_kg.py --type ir_qa --start 2025-01
python scripts/extract_kg.py --type meeting_materials --start 2025-01
python scripts/extract_kg.py --type inquiry_letters --start 2025-01
python scripts/extract_kg.py --type proposal_reference --start 2025-01
python scripts/extract_kg.py --type governance --start 2025-01

# 多类型一起跑
python scripts/extract_kg.py --types periodic_report esg_report ir_qa

# 从 data/report_new 批量抽取
python scripts/extract_batch_by_month.py --months 2025-01 2025-02 2025-03
```

### 查看进度

```bash
# 重新连到 screen 看日志
screen -r extract

# 查看结果条数
wc -l /root/riki_workspace/tag/result/2025-*/*/result_*.jsonl

# 查看数据文件数
find /root/riki_workspace/data -mindepth 3 -type f | awk -F/ '{print $5"/"$6}' | sort | uniq -c | sort -rn
```

## 日常操作

### 改代码

直接在宿主机编辑代码，重启容器生效：

```bash
vim /root/riki_workspace/tag/scripts/extract_kg.py
podman restart tag-extract
```

### 容器管理

```bash
# 查看容器状态
podman ps

# 查看日志
podman logs -f tag-extract

# 进容器调试
podman exec -it tag-extract bash

# 停止/启动
podman stop tag-extract
podman start tag-extract

# 重建容器（如改了挂载路径）
podman rm -f tag-extract
podman run -d --name tag-extract ...
```

### screen 管理

```bash
screen -ls           # 列出所有会话
screen -r extract    # 重连
# Ctrl+A, D          # 脱离
screen -X -S extract quit  # 关闭会话
```

## 附录：配置 Docker 国内镜像源

如果 `docker build` 拉不到基础镜像，编辑 `/etc/docker/daemon.json`：

```json
{
    "registry-mirrors": [
        "https://docker.1ms.run",
        "https://docker.xuanyuan.me",
        "https://docker.m.daocloud.io"
    ]
}
```

重启 Docker：

```bash
sudo systemctl restart docker
```

## 附录：vLLM 模型服务部署（如需在目标机新建）

```bash
# 安装
pip install -U vllm qwen-vl-utils==0.0.14

# 2 张 GPU 起 1 个实例
CUDA_VISIBLE_DEVICES=0,1 vllm serve /root/riki_workspace/models/Qwen3-VL-32B-Instruct-FP8 \
  --tensor-parallel-size 2 --port 8003 --host 0.0.0.0 \
  --max-model-len 131072 --served-model-name "Qwen/Qwen3-VL-32B-Instruct-FP8" \
  --limit-mm-per-prompt '{"video":0}' --mm-processor-cache-gb 0

# 4 张 GPU 起 2 个实例
MODEL_PATH=/root/riki_workspace/models/Qwen3-VL-32B-Instruct-FP8 ./serve/vllm_qwen32b.sh

# 用 screen 后台运行
screen -S vllm
CUDA_VISIBLE_DEVICES=0,1 vllm serve ...
# Ctrl+A, D 脱离
```
